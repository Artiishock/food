# Руководство по развертыванию проекта Food Slots

Этот документ содержит инструкции по развертыванию вашего проекта Food Slots в качестве постоянного веб-сайта. Проект состоит из двух основных частей: клиентской (фронтенд, созданный с использованием React и Vite) и серверной (бэкенд, созданный с использованием Node.js).

## 1. Развертывание фронтенда (клиентская часть)

Фронтенд проекта представляет собой статическое веб-приложение, которое можно развернуть на любой платформе для статического хостинга. Рекомендуемые платформы включают Vercel и Netlify благодаря их простоте использования и интеграции с GitHub.

### 1.1. Развертывание на Vercel

1.  **Создайте аккаунт Vercel:** Если у вас его нет, зарегистрируйтесь на [Vercel](https://vercel.com/). Вы можете войти через GitHub, GitLab или Bitbucket.
2.  **Импортируйте проект:** После входа в систему нажмите "New Project" и выберите свой репозиторий GitHub `Artiishock/food`.
3.  **Настройте проект:**
    *   **Root Directory:** Укажите `.` (корневая директория репозитория).
    *   **Build Command:** `pnpm build`
    *   **Output Directory:** `dist/public`
    *   **Install Command:** `pnpm install`
4.  **Развертывание:** Нажмите "Deploy". Vercel автоматически соберет и развернет ваше приложение. После завершения вы получите URL-адрес вашего развернутого фронтенда.

### 1.2. Развертывание на Netlify

1.  **Создайте аккаунт Netlify:** Если у вас его нет, зарегистрируйтесь на [Netlify](https://www.netlify.com/). Вы можете войти через GitHub.
2.  **Импортируйте проект:** На панели управления Netlify нажмите "Add new site" -> "Import an existing project" и выберите свой репозиторий GitHub `Artiishock/food`.
3.  **Настройте проект:**
    *   **Owner:** Выберите свой аккаунт GitHub.
    *   **Branch to deploy:** `main` (или любая другая ветка, которую вы хотите развернуть).
    *   **Base directory:** `.`
    *   **Build command:** `pnpm build`
    *   **Publish directory:** `dist/public`
4.  **Развертывание:** Нажмите "Deploy site". Netlify соберет и развернет ваше приложение. Вы получите URL-адрес вашего развернутого фронтенда.

## 2. Развертывание бэкенда (серверная часть)

Бэкенд проекта (Node.js) требует среды выполнения, которая может обрабатывать серверный код. Рекомендуемые платформы включают Render, Heroku или собственный VPS.

### 2.1. Развертывание на Render

1.  **Создайте аккаунт Render:** Если у вас его нет, зарегистрируйтесь на [Render](https://render.com/).
2.  **Создайте новый Web Service:** На панели управления Render нажмите "New" -> "Web Service".
3.  **Подключите репозиторий:** Подключите свой репозиторий GitHub `Artiishock/food`.
4.  **Настройте Web Service:**
    *   **Name:** Придумайте имя для вашего сервиса (например, `food-slots-backend`).
    *   **Root Directory:** `.`
    *   **Runtime:** `Node`
    *   **Build Command:** `pnpm install && pnpm build`
    *   **Start Command:** `pnpm start` (это запустит `NODE_ENV=production node dist/index.js`)
    *   **Region:** Выберите ближайший к вам регион.
    *   **Instance Type:** Выберите подходящий тип экземпляра (например, `Free` для тестирования).
5.  **Развертывание:** Нажмите "Create Web Service". Render автоматически соберет и развернет ваш бэкенд. Вы получите публичный URL-адрес для вашего бэкенда.

### 2.2. Настройка переменных окружения

Если ваш бэкенд или фронтенд используют переменные окружения (например, ключи API, URL-адреса баз данных), убедитесь, что вы настроили их на выбранной платформе хостинга. На Vercel, Netlify и Render это обычно делается через раздел "Environment Variables" в настройках проекта.

## 3. Подключение фронтенда к бэкенду

После развертывания фронтенда и бэкенда вам нужно будет настроить фронтенд для взаимодействия с развернутым бэкендом. Это обычно включает в себя обновление URL-адреса API в клиентском коде.

1.  **Получите URL бэкенда:** После развертывания бэкенда на Render (или другой платформе) вы получите публичный URL-адрес (например, `https://food-slots-backend.onrender.com`).
2.  **Обновите конфигурацию фронтенда:** Вам нужно будет найти место в клиентском коде, где определяется базовый URL-адрес API (например, в файле `src/config/api.ts` или `src/lib/utils.ts`, если такой существует, или напрямую в вызовах `axios`). Замените локальный URL-адрес (например, `http://localhost:3000`) на публичный URL-адрес вашего развернутого бэкенда.
3.  **Пересоберите и переразверните фронтенд:** После изменения URL-адреса API вам нужно будет пересобрать и переразвернуть фронтенд на Vercel/Netlify, чтобы изменения вступили в силу.

Следуя этим шагам, вы сможете успешно развернуть проект Food Slots в качестве постоянного веб-сайта.
